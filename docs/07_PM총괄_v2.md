## 1. 전체 직원 배치 및 모델 배정

| 직원 | 담당 | 모델 | 배정 근거 | Phase | 의존성 |
|------|------|------|-----------|-------|--------|
| **PM** | 총괄·조율·스펙 관리 | Opus | 복잡한 전략 판단, 파트 간 조율 | 전체 | — |
| **Claude-A** | 백엔드 API (FastAPI) | Sonnet | 코드 품질+속도 균형, 복잡한 API 설계 | 1 | 없음 (첫 시작) |
| **Claude-E** | 데이터 수집 엔진 (LS API) | Sonnet | LS API 연동, 실시간 데이터 파이프라인 | 1 | A와 병행 (DB 공유) |
| **Claude-C** | 프론트엔드 (Next.js) | Sonnet | UI/UX 구현, 반응형 디자인 | 1 | A와 병행 (Mock API) |
| **Claude-F** | 약관/FAQ/설문/면책조항 | Haiku | 단순 문서 작성, 빠른 반복 | 1 | 독립 |
| **Claude-B** | K-Trader 앱 (PyQt5) | Sonnet | 기존 코드 리팩토링, API 연동 | 2 | A Phase 1 완료 후 |
| **Claude-D** | AWS 인프라 자동화 | Sonnet | CDK/Terraform 복잡 설정 | 2 | A Phase 1 완료 후 |

### 모델 배정 근거
- **Opus**: PM 전용. 사업 전략, 문서 간 일관성 관리, 복합적 의사결정이 필요한 역할
- **Sonnet**: 코드 생성이 핵심인 역할. 품질과 속도 모두 필요. A/B/C/D/E 모두 해당
- **Haiku**: 약관/FAQ 등 구조화된 텍스트 작성. 복잡한 판단 불필요, 빠른 출력 우선

## 5. 파트 간 인터페이스 체크포인트

### Phase 1: Claude-E ↔ Claude-A 연결
```
□ LS API 수집 데이터 → Supabase 테이블 매핑 일치
□ leader_stocks 테이블 category 값 통일
□ supply_stocks 테이블 confirmed_at 시각 처리
□ 실시간 업데이트 주기 합의 (5초? 30초? 1분?)
```

### Phase 1: Claude-A ↔ Claude-C 연결
```
□ /api/v1/leaders/* 응답 포맷 일치
□ /api/v1/supply/* 유료 체크 로직 확인
□ 푸시 알림 토큰 등록/수신 흐름
□ 무료/유료 구분 UI 처리 방식
□ WebSocket vs REST 폴링 결정 (현재 주도주 실시간)
```

### Phase 2: Claude-A ↔ Claude-B 연결
```
□ /api/v1/license/verify 응답 포맷 일치
□ JWT 토큰 형식 합의
□ K-Trader에서 수급주 데이터 접근 방식
```

### Phase 2: Claude-A ↔ Claude-D 연결
```
□ /api/v1/cloud/deploy 입력 파라미터
□ cloud_deployments 테이블 컬럼명
□ AWS 자격증명 전달 방식
```

## 6. 리스크 목록

| 리스크 | 가능성 | 대응 |
|--------|--------|------|
| LS API에서 NXT 통합 데이터 미지원 | 중 | KRX/NXT 개별 조회 후 서버에서 합산 |
| 베타 유저 유료 전환 거부 | 중 | 할인 쿠폰 + 베타 기간 실적 데이터로 가치 증명 |
| Gemini 무료 API 한도 초과 | 낮 | 하루 1회 호출이라 초과 가능성 낮음. 초과 시 유료 전환 |
| 다중 계정 남용 (무료 체험 반복) | 중 | 구글 로그인 + 추가 인증으로 1인 1계정 |
| 수급주 정보 부정확 (잠정치 기반) | 중 | "14:30 잠정치 기준" 명시 + 면책 조항 |
| 키움/LS API 동시 사용 충돌 | 낮 | 완전 분리 (키움=K-Trader, LS=수집) |
| DART API 일일 한도 (1,000건) | 낮 | 수급주 종목만 조회하면 충분 |
| Stripe 한국 원화 설정 이슈 | 낮 | Phase 2 직전 사전 확인 |
| AWS 비용 초과 (Cloud 플랜) | 낮 | 사용자당 월 상한 설정 |
