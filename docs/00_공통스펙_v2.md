## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | Rich-Signal (웹/앱) + K-Trader (데스크톱 자동매매) |
| GitHub | https://github.com/SilverRain0716/Rich-signal |
| 목적 | 한국 주식 시장 정보 서비스 (주도주/섹터/수급주) + 조건식 기반 자동매매 SaaS |
| 핵심 원칙 | **조건식 로직은 절대 노출 금지** — K-Trader에만 내장 |

### 서비스 성격
- **Rich-Signal**: 시장 분석 정보 제공 서비스 (투자 자문 아님)
- **K-Trader**: 자동매매 프로그램 (조건식은 프로그램 내부에만 존재)
- 모든 페이지에 투자 면책 조항 삽입 필수

## 2. Phase 전략

| Phase | 내용 | 기간 |
|-------|------|------|
| **Phase 1 — Rich-Signal 베타** | 웹/앱 시장 정보 서비스 오픈 (전체 무료) | ~2개월 |
| **유료 전환** | 수급주 유료화 + 신규 회원 1주일 무료 체험 | 베타 종료 시 |
| **Phase 2 — K-Trader 출시** | K-Trader.exe 출시 + 전체 유료 플랜 활성화 | Phase 1 이후 |

### 베타 전략
- Phase 1 기간(1~2개월) 동안 모든 기능 무료 제공
- 유료 전환 시 기존 베타 회원에게 할인 쿠폰 제공
- 신규 가입 회원은 1주일 무료 체험 후 유료 전환
- 무료 체험 만료 시 유료 콘텐츠(수급주) 페이지 접근 차단 + 업그레이드 유도 메시지

## 7. 인프라 아키텍처

```
[수집 서버 — Windows AWS EC2]          [백엔드 서버 — Linux AWS EC2]
 LS XingAPI/Open-API                    Python FastAPI
  → 시세/거래대금/거래량/수급            → API 엔드포인트 제공
  → KRX + NXT 통합 데이터               → Gemini API 호출 (AI 예상 주도주)
 DART API → 공시 데이터                 → Firebase FCM 푸시 발송
 네이버 금융 → 뉴스 데이터              → Stripe 결제 처리
  ↓                                      ↕
  → Supabase DB (PostgreSQL) ←──────── API ←→ [유저 웹/앱]

[K-Trader 서버 — Windows AWS EC2]  ← 완전 별도
 키움 OpenAPI+ → 자동매매 전용
 (수집 서버와 절대 간섭 없음)
```

### 핵심 분리 원칙
- **수집 서버**: LS API 전용 (NXT 통합 데이터 수집)
- **K-Trader 서버**: 키움 API 전용 (자동매매)
- 키움 API는 동일 ID 중복 로그인 불가 → 반드시 분리
- 두 서버는 Supabase DB를 공유하여 연결

## 10. DB 스키마 (Supabase)

```sql
-- 사용자
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT UNIQUE NOT NULL,
  google_id TEXT,
  additional_auth TEXT,          -- 추가 인증 정보 (다중 계정 방지)
  stripe_customer_id TEXT,
  plan_type TEXT NOT NULL DEFAULT 'free'
    CHECK (plan_type IN ('free','signal_pro','ktrader_basic','ktrader_pro','ktrader_cloud')),
  is_beta_user BOOLEAN DEFAULT FALSE,
  free_trial_expires_at TIMESTAMPTZ,  -- 1주일 무료 체험 만료일
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 온보딩 설문
CREATE TABLE onboarding_surveys (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  experience_level TEXT,        -- 'beginner' | '1_3_years' | '3_plus_years'
  trading_style TEXT,           -- 'day' | 'swing' | 'mid_long'
  interest_sectors TEXT[],      -- ['semiconductor','bio','battery', ...]
  daily_invest_range TEXT,      -- 'under_1m' | '1m_10m' | '10m_50m' | '50m_plus'
  referral_source TEXT,         -- 'community' | 'youtube' | 'blog' | 'friend' | 'other'
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 탈퇴 설문
CREATE TABLE withdrawal_surveys (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID,
  reason TEXT,                  -- 'price' | 'inaccurate' | 'ux' | 'no_invest' | 'other'
  improvement TEXT,             -- 'more_stocks' | 'better_ui' | 'cheaper' | 'more_features' | 'other'
  rejoin_intent TEXT,           -- 'yes' | 'maybe' | 'no'
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 구독
CREATE TABLE subscriptions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  plan_type TEXT NOT NULL
    CHECK (plan_type IN ('signal_pro','ktrader_basic','ktrader_pro','ktrader_cloud')),
  stripe_subscription_id TEXT,
  status TEXT NOT NULL,         -- 'active' | 'canceled' | 'past_due' | 'trialing'
  current_period_end TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 주도주 이력
CREATE TABLE leader_stocks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  stock_code TEXT NOT NULL,
  stock_name TEXT NOT NULL,
  current_price NUMERIC,
  change_rate NUMERIC,
  volume BIGINT,
  trade_amount BIGINT,
  market TEXT CHECK (market IN ('KRX','NXT','INTEGRATED')),
  category TEXT NOT NULL,       -- 'previous' | 'ai_predicted' | 'current' | 'nxt_after' | 'nxt_pre'
  data_date DATE NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 섹터 순위
CREATE TABLE sector_rankings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  sector_name TEXT NOT NULL,
  change_rate NUMERIC,
  top_stocks TEXT[],
  data_date DATE NOT NULL,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 수급주
CREATE TABLE supply_stocks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  stock_code TEXT NOT NULL,
  stock_name TEXT NOT NULL,
  current_price NUMERIC,
  change_rate NUMERIC,
  foreign_net_buy BIGINT,
  institution_net_buy BIGINT,
  confirmed_at TIMESTAMPTZ,
  data_date DATE NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 종목 관련 뉴스
CREATE TABLE stock_news (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  stock_code TEXT NOT NULL,
  title TEXT NOT NULL,
  source TEXT,
  url TEXT,
  published_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 종목 관련 공시
CREATE TABLE stock_disclosures (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  stock_code TEXT NOT NULL,
  title TEXT NOT NULL,
  report_type TEXT,
  dart_url TEXT,
  published_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 라이선스/기기 (Phase 2)
CREATE TABLE devices (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  device_id TEXT NOT NULL,
  activated_at TIMESTAMPTZ DEFAULT NOW(),
  last_seen TIMESTAMPTZ
);

-- AWS 배포 (ktrader_cloud, Phase 2)
CREATE TABLE cloud_deployments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  ec2_instance_id TEXT,
  rds_endpoint TEXT,
  status TEXT CHECK (status IN ('pending','running','stopped','failed')),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- FCM 토큰
CREATE TABLE push_tokens (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  token TEXT NOT NULL,
  platform TEXT CHECK (platform IN ('web','android','ios')),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 11. 앱 페이지 구조

```
[시장] [주도주] [섹터] [수급주] [MY]
```

- 코스피/코스닥 지수
- 나스닥/다우존스
- 환율 (USD/KRW)
- 원유 (WTI)
- 비트코인 (BTC/KRW)
- 주도주/수급주 요약 카드 (각 페이지 진입점)

### 주도주 탭
**메인 뷰**:
- 전일 주도주 (KRX 정규장 기준)
- AI 예상 주도주 (Gemini 분석)
- 현재 주도주 (KRX+NXT 통합 거래대금 기준, 장중 실시간)

**한 뎁스 아래 — NXT 장외시간** (더보기/탭):
- 전일 애프터마켓 주도주 (15:30~20:00)
- 금일 프리마켓 주도주 (8:00~9:00)

### 섹터 탭
- 섹터별 상승률 순위 리스트
- 각 섹터 터치 시 해당 섹터 상위 종목 표시

## 12. 가입/탈퇴 설문 (객관식)

### 가입 시 온보딩 설문
```
Q1. 투자 경력은? → 초보 | 1~3년 | 3년 이상
Q2. 주요 매매 방식은? → 단타 | 스윙 | 중장기
Q3. 관심 섹터는? (복수 선택) → 반도체 | 바이오 | 2차전지 | 자동차 | IT/SW | 금융 | 기타
Q4. 일평균 투자 금액대는? → 100만원 미만 | 100만~1,000만원 | 1,000만~5,000만원 | 5,000만원 이상
Q5. 서비스를 알게 된 경로는? → 커뮤니티 | 유튜브 | 블로그 | 지인 추천 | 기타
```

### 탈퇴 시 설문
```
Q1. 탈퇴 사유는? → 가격이 비쌈 | 정보가 부정확 | 사용이 불편 | 더 이상 투자 안 함 | 기타
Q2. 개선 희망 사항은? → 더 많은 종목 | UI 개선 | 가격 인하 | 기능 추가 | 기타
Q3. 재가입 의향은? → 있음 | 아마도 | 없음
```

## 14. 보안 원칙 (전 파트 준수)

1. **조건식 로직은 코드/API/DB/로그 어디에도 노출 금지**
2. 키움 ID/PW는 K-Trader 클라이언트에서만 저장 (서버 전송 금지)
3. AWS 자격증명은 클라이언트에서만 저장
4. 모든 API는 JWT 인증 필수
5. 수급주 데이터는 유료 플랜 확인 후에만 응답
6. 다중 계정 방지: 구글 로그인 + 추가 인증 절차 적용
7. 모든 페이지에 투자 면책 조항 삽입
